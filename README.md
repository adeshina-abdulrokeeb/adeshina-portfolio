Adeshina Eniola's Portfolio Website:
1. A modern, responsive, and interactive portfolio built with HTML5, CSS3, and Vanilla JavaScript.  
2. Designed for speed, clarity, and precision - entirely framework-free.

Live Demo: 
1. https://adeshina-portfolio-omega.vercel.app/

---

Overview:
1. This portfolio is my professional space for showcasing projects and expertise across Frontend Development and Prompt Engineering.  
2. It is intentionally lightweight, focused on performance, accessibility, and real-world usability.
3. From an engaging hero section to a filterable project gallery, every component was crafted to communicate clarity, intent, and craft.

---

Motivation:
1. I built this site to communicate who I am as a developer; curious, intentional, and continually learning.  
2. It is designed to leave a lasting impression on collaborators, recruiters, and peers while serving as a practical, working portfolio.

---

Sections Breakdown:

A. Hero
1. Immediate brand presence including name, role, and clear call-to-action buttons.  
2. Subtle animations and a consistent dark theme set the tone for the rest of the site.

B. About
1. A concise professional narrative that explains my background, training, and motivation behind the work I do.

C. Skills
1. Highlights technical strengths in HTML5, CSS3, and JavaScript, as well as Git and general frontend development practices.

D. Prompt Engineering
1. A distinct section dedicated to my work in AI prompting - covering prompt frameworks, testing, and creative automation systems.

E. Projects
1. A filterable gallery divided into Frontend and Prompt Engineering categories.  
2. Each project card links to a live demo and GitHub repository.

F. Tools
1. A concise overview of the core tools and technologies that power my workflow.

---

Features:
1. Interactive preloader animation  
2. Scroll progress indicator and section-based navigation  
3. Smooth section reveals using the IntersectionObserver API  
4. Custom cursor trail and micro-interactions  
5. Responsive layouts using CSS Grid and Flexbox  
6. Accessible markup and semantic HTML structure  
7. Dark-only theme for consistent visual identity  
8. Lightweight and fully functional without dependencies

---

Tech Stack: Built completely from scratch using:
1. HTML5 – Semantic and accessible markup  
2. CSS3 – Grid, Flexbox, variables, animations, and transitions  
3. Vanilla JavaScript (ES6+) – DOM manipulation, event handling, and intersection observer  
4. No frameworks or UI libraries were used.
